# [알파벳(1987번)](https://www.acmicpc.net/problem/10989)

## 문제  
***첫째 줄에 수의 개수 N(1 ≤ N ≤ 10,000,000)이 주어진다. 둘째 줄부터 N개의 줄에는 숫자가 주어진다. 이 수는 10,000보다 작거나 같은 자연수이다.***

**제약조건**
시간제한 : 3초,  메모리 제한 : 8MB

## 풀이
정렬 문제라고 쉽게 생각하다가 큰코다친 문제이다.  
처음에는 map을 이용하여 풀었다. 그러나, 메모리초과가 난다. key, value 각각 int타입이므로 합산 8바이트, 최대 10,000의 숫자를 포함하므로 8MB안에서는
충분히 해결할 수 있을 것이라고 생각했었다.  
두 번째로 배열을 이용한 counting sort를 사용해 보았다. 하지만, 이번에는 시간초과가 난다.(ㅎㅎㅎㅎ...)  
해결방안을 찾던 중, 입출력을 cin,cout이 아닌, scanf,printf를 이용하면 기존 C++ 입출력보다 3배의 효율을 가질 수 있다는 [정보](https://stackoverflow.com/questions/1042110/using-scanf-in-c-programs-is-faster-than-using-cin)를 찾았다.  
결과는 2480ms의 시간으로 아슬아슬하게 통과했다. 좀 더 빠르게 해결할 수 있는 다른 방법이 없을까 고민해 볼 필요가 있는 문제인 것 같다.
